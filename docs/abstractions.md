# Key Abstractions (Reference Catalog)

Exhaustive list of every abstraction in the codebase, grouped by concern. For conceptual architecture and design rationale see `architecture.md`.

## Core Runtime

- **Provider** â€” LLM backend (Claude, GPT, Gemini, local). Swappable per agent. `core/provider.py`
- **Agent** â€” Autonomous unit with a role, tools, and a provider. Stateless between tasks. `core/agent.py`
- **Skill** â€” Reusable capability with middleware chain (retry, logging, timeout). Provider-independent. `core/skill.py`
- **Orchestrator** â€” Coordinates agents, task decomposition, anti-stall enforcement. `core/orchestrator.py`
- **Cooperation** â€” Inter-agent messaging: delegation, results, conflict resolution. `core/cooperation.py`. Wire-format messages (`DelegateMessage`, `ResultMessage`, `ConflictMessage`, `CapabilityQueryMessage`, `CapabilityResponseMessage`) live in `core/cooperation_messages.py` with a `parse_message` dispatcher. Spec, state machine, and error semantics: [docs/cooperation-protocol.md](cooperation-protocol.md).
- **OrchestratorClient** â€” Embedded Python client (`client.py`). Wraps Orchestrator, Agent, SkillRegistry, and StateGraph into a single API. Supports `run_agent()`, `run_team()`, `run_graph()`, `list_agents()`, `list_skills()`, plus sync wrappers. No HTTP server required.

## Routing & Cost Control

- **TaskRouter** â€” Smart routing: 6 strategies (local-first, cost-optimized, complexity-based, etc.). Category-aware: auto-detects task domain (finance, data-science, marketing, software) and selects appropriate agents. Fallback routing uses category-matched agents instead of defaulting to backend+frontend. `core/router.py`
- **UsageTracker** â€” Cost tracking with budget enforcement (per task/session/day). `core/usage.py`
- **HealthMonitor** â€” Provider health: latency, error rates, availability, auto-failover. `core/health.py`
- **RateLimiter** â€” Per-provider rate limiting. `core/rate_limiter.py`
- **ProviderPresetManager** â€” One-click presets: local_only, cloud_only, hybrid, high_quality. `core/provider_presets.py`

## Observability

- **AuditLog** â€” Structured audit trail: 11 event types, filtering, task traces. `core/audit.py`
- **MetricsRegistry** â€” Prometheus-compatible metrics (counters, gauges, histograms). `core/metrics.py`
- **Tracing** â€” Optional OpenTelemetry integration. Initialized in `server.py` at startup via `setup_tracing()` + `instrument_fastapi()`. Spans on `Provider.traced_complete()`, `Agent._execute_with_provider()`, graph nodes. `instrument.py` also feeds `tracing_metrics` collectors (LLM durations, node durations, stall counts) which are exported at `/metrics` for Prometheus. Graceful no-op when OTel packages not installed. Exports via OTLP HTTP to Tempo.
- **AlertHandler** â€” Receives Grafana webhook alerts, collects diagnostics (recent errors, usage, metrics), creates GitHub issues with `gh` CLI. Triggers automated root-cause analysis via `.github/workflows/alert-analysis.yml`.

## Graph Engine

- **StreamEvent / astream()** â€” Real-time graph execution streaming. `CompiledGraph.astream()` yields `StreamEvent` at each node start/end/error, with state deltas and timing. `core/graph.py`
- **GraphTemplateStore** â€” Versioned graph templates with JSON serialisation and `build_graph()`. `core/graph_templates.py`
- **SubGraphNode** â€” Wrap compiled graphs as callable nodes with I/O mapping. `core/graph_patterns.py`
- **Hybrid Graph Execution** (PR #84) â€” `CompiledGraph.invoke(preload=[...], store=...)`. Pre-fetches `(namespace, key, state_key)` triples from a `BaseStore` and merges into `initial_state` before graph traversal. Lets nodes reference external memory without extra skill calls. Missing keys silently skipped. Regression-safe.
- **RunManager (SSE)** â€” HTTP SSE streaming for graph execution. Creates background runs, fans events to multiple SSE subscribers, supports HITL interrupt/resume with configurable timeout. Max 100 runs, 30-min TTL eviction. Mirrors events to EventBus for WebSocket clients. `dashboard/sse.py`

## Memory & State

- **BaseStore** â€” Cross-thread persistent key-value store (namespace, filter, TTL). Separate from checkpoints. `core/store.py`
- **PostgresStore** â€” Durable PostgreSQL backend for BaseStore. JSONB values, lazy TTL expiry, UPSERT semantics, dot-encoded namespaces. Wired into dashboard at startup when `DATABASE_URL` is set; falls back to InMemoryStore. `core/store_postgres.py`
- **SessionStore** â€” Session-scoped wrapper on BaseStore. Auto-tracks written keys, deletes all session data on close(). Async context manager.
- **ConversationManager** â€” Thread-based multi-turn memory. Accumulates messages across invocations via checkpointing. Supports fork, clear, max_history trim. Persists to PostgreSQL and survives container restarts. Sessions can be restored from job records via `POST /api/jobs/{session_id}/restore`. Supports **configurable context summarization** via `SummarizationConfig`: when threads exceed a trigger threshold (message count, token count, or fraction of max_history), older messages are replaced with a single system summary, retaining the last N messages verbatim. Metrics: `conversation_summarization_total` counter, `conversation_tokens_saved` gauge. `core/conversation.py`
- **MemoryFilter** â€” Sanitizes session-scoped file paths (job dirs, tmp files, uploads, workspace) before persisting to conversation memory or cross-thread store. Replaces paths with `[session-file]` placeholder. Messages containing only session-file references are dropped. Integrated with `ConversationManager._save_thread()` and `InMemoryStore.aput()`. `core/memory_filter.py`
- **Hierarchical Namespace Helpers** (PR #81) â€” `core/store.path_to_namespace`, `namespace_to_path`, `descends_from`, `namespace_depth`, `NAMESPACE_SEP` (default `.`). `BaseStore` gains `aget_path(path, key)`, `aput_path(path, key, value)`, `asearch_path(path_prefix)` for dotted-path ergonomics.
- **Verbatim Checkpoint Log** (PR #81) â€” `Checkpoint` dataclass has optional `raw_log: str | None`. Persisted in `InMemoryCheckpointer`, `SQLiteCheckpointer`, and `PostgresCheckpointer` (new `raw_log TEXT` column, auto-migrated via `ADD COLUMN IF NOT EXISTS`). Opt-in â€” defaults to None to keep storage flat.
- **PersonalizedMemory** (P4) â€” User-scoped memory facade over `BaseStore`. Namespace convention: `("user", user_id)`. Methods: `put(user_id, key, value, ttl_seconds)`, `get(user_id, key)`, `list(user_id, limit=50)` (newest first), `delete(user_id, key) -> bool`, `wipe(user_id) -> int` (GDPR erasure). Applies `MemoryFilter` to every write so session artefacts never enter long-term memory. Used by `Agent` to inject a `<user_profile>` block into the system prompt when both `personalized_memory` and `user_id` are provided at construction. The block is populated by calling `Agent.prefetch_user_profile()` before execution. HTTP API at `/api/user-memory/users/*`. `core/personalized_memory.py`

## Skills & Middleware

- **ProfileExtractorSkill** (P4) â€” Memory-category skill (`name = "extract_user_profile"`). Accepts `{user_id, recent_messages}`. Calls an injected `Provider` to summarise preferences, style notes, and recurring topics into `{preferences, style_notes, recurring_topics}`, then persists via `PersonalizedMemory.put(user_id, "profile", value)`. Best-effort: provider errors and JSON parse failures return `SkillResult(success=False)` without raising. `skills/profile_extractor_skill.py`
- **SkillMiddleware** â€” Composable interceptors on skill execution (retry, logging, timeout, cache). `core/skill.py`
- **Tool Description (`_description`)** â€” Optional `_description` parameter on every tool call. Extracted before execution (never forwarded to the skill), logged, propagated via `SkillRequest.metadata["tool_description"]`, included in `AuditEntry.tool_description`, and shown in dashboard tool-call events. Injected into `to_tool_definitions()` schemas so LLMs can explain why they invoke a tool.
- **LLM Cache** â€” Shared `InMemoryCache` for LLM node responses. Activated via `cache_policy` param on `llm_node()`. Skips cache when `temperature > 0`. Dashboard shows hits/misses/rate in real time.
- **Tool Cache** â€” `cache_middleware()` on `SkillRegistry` caches idempotent skills (`file_read`, `glob_search`). Auto-invalidates on `file_write`.
- **Progressive Skill Loading** â€” System prompts include only compact `SkillSummary` (name + description + category) instead of full instructions. Agents invoke `load_skill` to fetch detailed instructions on demand, reducing base prompt token usage. `skill_loads_total` counter tracks load frequency.
- **Verification Gate Middleware** (PR #59) â€” `core/skill.verification_middleware(validators, metrics=)`. Rejects skill results that fail the per-skill validator (returning `False` or `(False, reason)`). Converts failures into error `SkillResult`. Metrics: `verification_total`, `verification_pass_total`, `verification_fail_total`, `verification_duration_seconds` (all per-skill). See `docs/phase2.md`.
- **Workspace VerificationGate** (v1.5 P1, Phase 2) â€” `core/verification_gate.VerificationGate(verifiers, *, fail_fast=True, emit_event=None)`. Workspace-level (distinct from the per-skill middleware above): runs a chain of `WorkspaceVerifier` protocol implementations against a produced workdir after `run_team()`, returns a `VerificationReport` with stable per-failure `signature` for cross-attempt dedup. Cost-first ordering; warnings never trip fail-fast. Bundled verifiers in `core/verifiers/` (extended through Phase 7.7): `SyntaxVerifier` (`py_compile` + `json.loads`), `EncodingVerifier` (literal-`\\n` corruption heuristic â€” catches the 2026-05-16 `frontend/package.json` failure mode), `DependencyVerifier` (static check against `KNOWN_BAD_PINS`, e.g. `psycopg<3` â†’ suggest `psycopg2-binary` â€” catches the 2026-05-16 dep-typo failure mode), `ImportVerifier` (AST-walks every `*.py`, drops stdlib + local modules, flags every top-level import without a matching `requirements*.txt` / `pyproject.toml` entry â€” catches the 2026-05-16(b) `passlib`/`python-jose` missing-dep failure mode), `WorkspaceCoherenceVerifier` (cross-file rule: `docker-compose.yml::services.*.environment.DATABASE_URL` scheme must match the default in `*.py` `os.getenv("DATABASE_URL", ...)` â€” catches iter-3 silent contradictions from 2026-05-16(b)), `RuntimeSmokeVerifier` (Phase 7.7 ground-truth tier â€” `cost_estimate_s=30`; **Phase 7.9c**: cache is keyed by the SHA-256 of the *canonical normalised package set* â€” version pins, comments and whitespace edits no longer invalidate the cache; on miss, scans sibling cached venvs for a STRICT-SUBSET match and clones it via `cp -a --reflink=auto` + `pip install <delta>`, cutting iterative-update cost from ~30-90 s to ~3-10 s; runs `pip install` + `python -c "import X"` for every top-level local module; emits failures with `category="missing_dep"` so the existing `requirements_append` auto-fix matches without a new pattern), `EntrypointVerifier` (Phase 7.11a â€” `cost_estimate_s=20`; detects the production entrypoint by reading `docker-compose.yml::services.*.command` or `Dockerfile` `CMD`, then launches the uvicorn-shaped command on a free port using the smoke-cached venv with a hard 12 s timeout + health probe; surfaces `entrypoint_crash` / `entrypoint_timeout` failures with the captured stderr tail â€” catches relative-import bugs and startup-time crashes that bare-import probes miss, e.g. the 2026-05-16(g) `from .database` regression under `uvicorn main:app`), `E2ESmokeVerifier` (Phase 7.11b â€” `cost_estimate_s=60`, **opt-in via** `REPAIR_LOOP_E2E_ENABLED=true`; serves the detected frontend dir (`frontend/`, `web/`, etc.) on a tmp HTTP server, optionally launches the backend on a sibling port, opens the page with headless Chromium via Playwright, captures JS console errors + failed network requests + canvas presence; soft-dependency on Playwright â€” emits a `warning` when the env var is on but the package is missing instead of crashing the gate â€” catches the entire frontend/backend integration class, e.g. the 2026-05-16(g) `data.forEach` on a `{satellites: [...]}` shape mismatch). Events: `verification.started / verifier.started / verifier.finished / verification.finished`. Consumed by the `RepairLoop` (Phase 3). See `docs/architecture-repair-loop.md` and `tests/test_verification_gate.py` + `tests/test_verifiers.py`.
- **RepairLoop** (v1.5 P1, Phase 3 â€” post-condition guard added in Phase 7.7, `max_wall_s` cap added in Phase 7.9a) â€” `core/repair_loop.RepairLoop(team_runner, gate, *, max_attempts=5, max_cost_usd=0.50, max_wall_s=1800, emit_event=None, pattern_registry=None)`. Wraps any async `team_runner(task, **kw) -> TeamRunResult` with verify-and-retry semantics: invokes the team, asks the gate to verify the workdir, on failure builds a structured retry prompt (top-3 failures + past-attempt history + optional file excerpts) and re-invokes the team up to `max_attempts` times (default 5). Hard cost cap aborts as soon as the cumulative cost exceeds `max_cost_usd`. Signature memory detects "same failure repeated" and triggers an escalation strategy (`more_context` â€” re-prompt with file excerpts). Optional `pattern_registry` short-circuits known failures deterministically without an LLM call (Phase 4). **Post-condition guard**: `_try_auto_fix` snapshots `(path, original_bytes)` for every file an action touches, re-verifies once, and if the new report has strictly more failures than the pre-fix one it reverts every touched file and emits `repair.auto_fix_reverted` (prevents a buggy pattern from compounding the failure â€” e.g. the 2026-05-16(d) regression where the auto-fix added bare `psycopg2`). Result statuses: `passed / partial / aborted_budget / aborted_cost / aborted_time`. Events: `repair.started / attempt_started / attempt_finished / escalated / auto_fixed / auto_fix_reverted / aborted{reason: cost|time} / finished`. See `docs/architecture-repair-loop.md` and `tests/test_repair_loop.py`.
- **FailurePatternRegistry** (v1.5 P1, Phase 4 â€” extended in Phase 7 follow-up) â€” `core/failure_patterns.FailurePatternRegistry` loaded from `core/failure_patterns.yaml` via `load_default_registry()`. Each `FailurePattern` matches a `VerifierFailure.category` + a regex over `.message`, then runs an in-process auto-fix action: `pip_pin_repair` (rewrites `requirements*.txt` lines per a `replacements` map, e.g. `psycopg` â†’ `psycopg2-binary>=2.9`), `unicode_unescape` (decodes over-escaped newlines back to real characters; same heuristic as `EncodingVerifier`, defence-in-depth), `requirements_append` (appends a missing package to the nearest `requirements*.txt`, parsed out of an `ImportVerifier` failure's `detail` â€” knows about common module/package aliases such as `jose` â†’ `python-jose`, `bs4` â†’ `beautifulsoup4`), or `noop` (records the match but does nothing â€” surfaces known-but-unfixable failures in the dashboard). Patterns are matched in YAML order; first hit wins. Adding a new pattern is a YAML edit â€” no Python change. Hooked into `RepairLoop` via its `pattern_registry=` argument: the loop calls `registry.apply(failure, workdir)` before each LLM retry, so a known failure costs zero LLM calls. See `core/failure_patterns.yaml` and `tests/test_failure_patterns.py`.
- **Repair-loop wiring** (v1.5 P1, Phase 5 â€” ON by default since Phase 7) â€” `dashboard/agent_runtime_router._run_team_with_repair`. Drop-in wrapper around `run_team()`, active unless `REPAIR_LOOP_ENABLED=false`. Tunable via `REPAIR_LOOP_MAX_ATTEMPTS` (default `5`) and `REPAIR_LOOP_MAX_COST_USD` (default `0.50`). Builds a `RepairLoop` per call with the **six** bundled verifiers (`Syntax`, `Encoding`, `Dependency`, `Import`, `WorkspaceCoherence`, `RuntimeSmoke` â€” last one added in Phase 7.7 as the ground-truth tier) and the default `FailurePatternRegistry`, then bridges its sync `emit_event(name, data)` sink to the async `EventBus` via `asyncio.create_task`. Return shape: the original `run_team()` dict (so `output`, `plan`, `agent_costs`, `total_tokens` flow unchanged to the dashboard / `job_logger` / `usage_db`), with `success` re-evaluated as `run_team.success AND verifiers.passed` and a new top-level `repair: {status, attempts, cumulative_cost_usd, final_passed, final_failures[], auto_fixed_signatures[]}` block â€” surfaced in the React dashboard as a `Repair loop: âœ“ passed (1 attempt)` / `âš  partial` / `âœ— aborted` system message after the team-complete event (`frontend/src/hooks/useWebSocket.ts`, `frontend/src/api/types.ts::RepairSummary`). See `tests/test_repair_loop_wiring.py` and `docs/architecture-repair-loop.md` Â§ "Wiring into `team_run` (Phase 5)".
- **Context Loader Middleware** (PR #61) â€” `core/skill.context_loader_middleware(context_dir, target_skills=, metadata_key=, max_bytes=, metrics=)`. Reads `*.md` files from a directory, caches concatenated content, injects under `request.metadata[metadata_key]`. Skills opt-in by reading that key. Metrics: `context_files_loaded_total`, `context_bytes_injected` (per skill).

## Prompt Engineering

- **Marker-based Prompt Injection** (PR #57) â€” `core/prompt_markers.py`. `inject_marker_sections(base, {marker: content})` inserts/replaces named blocks delimited by `<!-- NAME START -->` / `<!-- NAME END -->` comments; idempotent, never mutates inputs. `Agent.set_prompt_section(marker, content)` updates instance state and increments counter `marker_updates_total{agent=<name>}`. `Agent.build_system_prompt()` returns the effective prompt with all markers applied â€” used at every provider call. See `tests/test_prompt_markers.py` and `docs/prompt-engineering.md`.
- **PromptRegistry** (PR #56) â€” `core/prompt_registry.py`. Tag-indexed, metadata-rich prompt template store backed by `BaseStore` (durable via `PostgresStore`). `PromptTemplate(name, content, tags, category, version, description, metadata)` dataclass with `.format(**kwargs)`. Namespace: `("prompt",)`. API: `register`, `get`, `delete`, `search(tags=, category=)` (AND-intersection on tags), `list_all`. Metrics: `prompt_registry_lookups_total`, `prompt_registry_hits_total`, `prompt_registry_misses_total`, `prompt_registry_lookup_duration_seconds`. REST endpoints at `/api/prompts*`. Wired at startup in `dashboard/app.py`. Frontend: dedicated **Prompts** floating-action panel. See `tests/test_prompt_registry.py`.
- **Compaction Metrics** (PR #60) â€” `ConversationManager` now accepts `metrics: MetricsRegistry`. Every `summarize_thread` call records counter `conversation_summarization_total`, gauge `conversation_tokens_saved` (cumulative), histogram `conversation_summarization_duration_seconds`, gauge `conversation_compaction_ratio` (last run), counter `conversation_messages_compacted_total`. Endpoint: `GET /api/compaction/stats`. Header widget **Tokens saved** in the dashboard surfaces the savings live. See `tests/test_compaction_metrics.py`.

## Reliability & Safety

- **ToolRecovery** â€” Detects dangling tool calls (assistant messages with `tool_calls` that have no matching `ToolMessage` response) and injects placeholder responses. Called automatically in `Agent.execute()` before each LLM call and in `ConversationManager._load_thread()` when restoring persisted threads. `core/tool_recovery.py`
- **Guardrail / GuardrailManager (P3)** â€” Pluggable pre/post LLM-call safety layer. `Guardrail` ABC with `check_input(messages) -> GuardrailResult` and `check_output(response) -> GuardrailResult`. `GuardrailResult(passed, reason, action, redacted_text)` is a frozen dataclass. `GuardrailManager` evaluates an ordered list: short-circuits on the first `block`, aggregates `redact` results. `GuardrailBlocked(RuntimeError)` is raised by `Agent.execute()` on a block so the orchestrator can surface a clean error. Built-ins: `PIIScanner` (email, phone, SSN, IBAN, credit card â€” default redact), `SecretsScanner` (AWS keys, GitHub tokens, generic API keys â€” default block), `PromptInjectionDetector` (heuristic phrase matching â€” default block), `OutputSchemaGuard(schema)` (JSON Schema validation â€” default block), `CostGuard(budget_usd, get_current_cost)` (pre-call budget check â€” default block). `guardrail_manager_from_config(dict)` converts the `guardrails:` section of `orchestrator.yaml` into a configured manager. Events emitted via injected callable; metrics: `guardrail_checks_total{type,side}`, `guardrail_blocks_total{type}`, `guardrail_redactions_total{type}`. `core/guardrails.py`.
- **LoopDetector** â€” Per-session sliding window loop detection for agent tool calls. Hashes tool_name+params (MD5), tracks in a `deque(maxlen=20)`. Warns at 3 repeats, hard stops at 5. LRU eviction at 500 sessions. Integrated into `Agent.execute()` via optional `loop_detector` + `session_id` params. Emits `loop.warning` / `loop.hard_stop` events; increments `loop_warnings_total` / `loop_hard_stops_total` counters.
- **Resilience** â€” `core/resilience.py`. `RetryPolicy` (exponential backoff with jitter, customizable `retryable` predicate) and `CircuitBreaker` (closed/open/half_open, threshold + cooldown). Combine via `resilient_call()`. Opt-in per-call wrapping â€” does not force behavior on existing providers. Non-retryable by default: `ValueError`, `TypeError`, `CircuitOpenError`.
- **SmokeTester** â€” `core/smoke_tester.py`. Detects a project's language (20 languages) from config files + conventional entry-point filenames, then runs a deps-free syntax check (`python -m py_compile`, `cargo check`, `node --check`, `go vet`, `bash -n`, `php -l`, `ruby -c`, etc.). Never raises â€” gracefully skips when the toolchain is missing (`shutil.which` guard). Wired into `run_team()` between the validation step and the summary step: a failure prepends a structured re-assignment (`{agent, task}`) so the broken entry point is fixed before summary is produced. Disable globally with `DISABLE_SMOKE_TEST=true`. Result exposed on the team-run return dict under `smoke_test`. See `tests/test_smoke_tester.py`.
- **Atomic Task Validator** (PR #59) â€” `core/atomic_tasks.validate_atomic_tasks(assignments)`. Lints team-lead decompositions for tasks that are too long, contain too many imperatives, or have multi-step conjunctions. Wired into `dashboard/agent_runner.run_team`; issues emit as `agent.step` events (no hard gate). Counter `tasks_rejected_too_complex_total`. See `tests/test_phase2.py`.
- **Modality Detection** (PR #88) â€” `core/modality.py`. `detect_modality(task_input)` returns one of `Modality.{TEXT, CODE, IMAGE, STRUCTURED, EQUATION, MIXED}`. Deterministic rule-based: detects PNG/JPEG/GIF/WebP magic bytes, dicts with image fields, LaTeX equations, code patterns (fenced blocks, function definitions, shebangs, Java/C signatures), structured tabular data. Priority: IMAGE > MIXED > STRUCTURED > EQUATION > CODE > TEXT. Counter `modality_detected_total{modality=...}` via `record_detection(modality, metrics)`. See `docs/phase3.md`.

## MCP (Model Context Protocol)

## Sandbox

- **Sandbox** â€” Isolated execution environment (Docker or local). `SandboxConfig` controls image, timeout, memory/CPU limits, network, writable paths, `exposed_ports` (port forwarding), `startup_command`, and `env_vars`. `PortMapping` maps container ports to host ports (auto-assign or explicit). `SandboxInfo` provides runtime introspection (status, mapped ports, uptime). Virtual path mapping with traversal protection. `SandboxedShellSkill` wraps sandbox as a drop-in Skill for agent use. `core/sandbox.py`
- **SandboxManager** â€” Session-scoped sandbox lifecycle in dashboard. Lazy initialization on first use, per-session workspace isolation (`/workspace/{session_id}/`), configurable `max_concurrent` (default 10), LRU eviction, cleanup on shutdown. **Port allocation pool** (default range 9000-9099) prevents host-port collisions between sessions. `get_sandbox_info(session_id)` returns `SandboxInfo` with live container metadata. Enabled via `SANDBOX_ENABLED=true`. Env vars: `SANDBOX_TYPE` (docker/local), `SANDBOX_IMAGE`, `SANDBOX_TIMEOUT`, `SANDBOX_MEMORY`, `SANDBOX_MAX_CONCURRENT`. API: `GET /api/sandbox/status`, `GET /api/sandbox/{session_id}/info` (ports, status, uptime), `GET /api/sandbox/{session_id}/logs` (SSE log streaming), `DELETE /api/sandbox/{session_id}`, `WS /ws/sandbox/{session_id}/terminal` (interactive shell via xterm.js).
- **Sandbox Live Stats** (PR #81 follow-up) â€” `Sandbox.get_stats()` queries `docker stats --no-stream` and returns CPU %, memory bytes/limit/percent, net rx/tx. Endpoint `GET /api/sandbox/{session}/stats`. Frontend: live CPU/MEM sparklines (30 samples at 3 s) in the Sandbox **Status** tab.

## Integrations & I/O

- **DocumentConverter** â€” Converts uploaded files (PDF, Excel, CSV, Word, PowerPoint, HTML, text) to Markdown for LLM consumption. Graceful fallback when optional deps missing. Limits: 10 MB file size, 50 PDF pages, 10,000 spreadsheet rows. Upload via `POST /api/upload` (multipart/form-data). `core/document_converter.py`
- **ClarificationManager** â€” Structured agent-human clarification. 5 typed request categories (missing_info, ambiguous, approach, risk, suggestion). Blocking mode pauses agent until response or 5-minute timeout. Non-blocking mode emits event and continues. Events: `clarification.request`, `clarification.response`, `clarification.timeout`. `core/clarification.py`
- **TelegramBot** â€” Telegram integration using long-polling (no public IP required). Maps Telegram chats to conversation_ids and routes free-text to agents. ## Quality

- **Evaluator Framework (P2)** â€” LLM-judge + rubric-based quality measurement for agents. `core/evaluator.py`
  - `EvalCase` / `EvalRun` / `EvalScore` / `EvalReport` â€” frozen dataclasses carrying the scenario, the agent response, individual grades, and the aggregated report.
  - `Evaluator` (ABC) â€” single-method interface: `async evaluate(case, run) -> EvalScore`. Open/closed: add new strategies without modifying existing code.
  - `RubricEvaluator` â€” deterministic checks over agent output: `contains`, `not_contains`, `regex`, `json_schema`, `max_length`, `min_length`. Checks carry optional `weight`; score is the weighted pass fraction.
  - `LLMJudge` â€” delegates scoring to a strong `Provider` (DIP: accepts the abstract `Provider`, never a concrete class). Prompts the model with a structured rubric template and expects `{"passed": bool, "score": float, "detail": str}`. Robust JSON extraction: strips code fences, falls back to regex brace-matching, never crashes on malformed output.
  - `EvalSuite` â€” binds cases + evaluators + an agent callable. `async run(agent_callable)` executes all cases, applies every evaluator, and computes summary metrics: `pass_rate`, `mean_score`, per-evaluator breakdowns.
  - `JsonDataset` â€” loads `EvalCase` lists from a JSON or YAML file (`{"cases": [...]}`). Validates `prompt` (required), auto-assigns `case_id` from index if absent. Raises `ValueError` on structural errors.
  - **CLI runner** â€” `python -m evals.runners.cli --suite evals/datasets/smoke.json --agent team-lead --provider openrouter --model openai/gpt-4o`. Coloured summary table; `--dry-run` mode for smoke-testing suites without LLM calls; `--json` for machine output.
  - **Bundled dataset** â€” `evals/datasets/smoke.json` â€” 5 hand-picked cases: code summary, math reasoning, JSON output, safety refusal, conversational.
  - **REST API** â€” `dashboard/evals_routes.py` (`/api/evals/run`, `/api/evals/runs`, `/api/evals/runs/{id}`, `/api/evals/compare?a=&b=`). Background execution, in-memory report store (last 50), side-by-side delta comparison.

## Responsive Layout

- **Breakpoint constants** (`frontend/src/lib/breakpoints.ts`) â€” `BP` object (`mobile: 768`, `tablet: 1024`, `desktop: 1280`) plus pre-built `mq.*` media-query strings. Single source of truth for both TS and (progressively) CSS.
- **`useBreakpoint()` hook** (`frontend/src/hooks/useBreakpoint.ts`) â€” matchMedia-based, SSR-safe, reactive. Returns `{ isMobile, isTablet, isDesktop, reducedMotion }`. Replaces direct `window.innerWidth` reads.
- **Responsive lint** (`scripts/check_responsive.sh`) â€” CI guard that blocks new hardcoded breakpoints and stray viewport reads in `frontend/src/`. Baseline file (`frontend/.responsive-baseline.txt`) pins pre-existing exceptions; rebuild via `scripts/check_responsive.sh --update-baseline`.
- **Cross-viewport smoke E2E** (`frontend/e2e/chat-smoke.spec.ts`) â€” Playwright spec that runs once per viewport (375Ã—667 mobile + 1440Ã—900 desktop, both Chromium) against the dev bundle with all `/api/*` mocked. CI job: `frontend-e2e` in `deploy.yml`.
- **iOS safe-area** â€” `viewport-fit=cover` in `frontend/index.html` + `env(safe-area-inset-*)` padding on `.app-header` and `.chat-input` in `frontend/src/index.css` so iPhones with notches / dynamic islands don't clip header chrome or the bottom action bar.

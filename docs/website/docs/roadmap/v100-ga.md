# v1.0.0 — General Availability

The product is ready for public use.

| ID | Feature | Files | Detail |
|----|---------|-------|--------|
| GA-01 | Stable API | `dashboard/app.py` | Versioned REST API (`/api/v1/`) with OpenAPI spec, no breaking changes |
| GA-02 | pip installable | `pyproject.toml` | `pip install agent-orchestrator` — publish to PyPI |
| GA-03 | Web-based config | `dashboard/static/`, `dashboard/app.py` | Edit agents, skills, graphs, provider settings from the dashboard UI |
| GA-04 | Multi-project support | `core/project.py` (new) | Manage multiple codebases from one dashboard instance |
| GA-05 | User management | `core/users.py` (new), `dashboard/auth.py` | Multi-user with roles: admin, developer, viewer |
| GA-06 | Provider presets | `dashboard/app.py`, `dashboard/static/` | One-click setup: "local only", "cloud only", "hybrid" mode |
| GA-07 | Documentation site | `docs/website/` | Full docs with tutorials, API reference, examples (this site) |
| GA-08 | Migration wizard | `core/migration.py` (new) | Import from LangGraph / CrewAI / AutoGen configs |

## Definition of Done

v1.0.0 is ready when:

- All features from v0.4.0 through v0.8.0 are implemented and tested
- API is versioned and documented (OpenAPI)
- Package is on PyPI
- Dashboard allows full configuration without editing files
- At least 3 users can work on separate projects simultaneously
- Documentation site is complete with getting started guide
- 90%+ test coverage on core modules

## What Comes After

After v1.0 ships, evaluate:

1. **Is there revenue?** If users are paying > 100 EUR/month total, consider infrastructure scaling
2. **Is there demand for enterprise features?** SSO, audit logs, SLA
3. **Is fine-tuning needed?** If domain-specific quality matters, invest in GPU infra

See [Post-MVP Scaling](./post-mvp-scaling) for the infrastructure expansion plan.

# Deployment Guide

How to deploy the Agent Orchestrator to production.

| Domain | Service |
|--------|---------|
| `https://agents-orchestrator.com` | Dashboard (FastAPI + WebSocket) |
| `https://monitoring.agents-orchestrator.com` | Grafana (metrics dashboards) |

## Architecture

```
Browser
  ├── agents-orchestrator.com:443
  │     → Nginx (SSL, rate limiting, security headers)
  │       → Dashboard :5005 (FastAPI + WebSocket)
  │         → PostgreSQL :5432
  │         → Redis :6379
  │
  └── monitoring.agents-orchestrator.com:443
        → Nginx (SSL, security headers)
          → Grafana :3000
            → Prometheus :9090
```

## Prerequisites

| Requirement | Details |
|-------------|---------|
| AWS EC2 | Amazon Linux 2023 or Ubuntu 22.04, t3.small+ |
| Elastic IP | Static public IP associated to the EC2 instance |
| DNS A records | `agents-orchestrator.com` + `monitoring.agents-orchestrator.com` → Elastic IP |
| Security Group | TCP 22 (SSH, your IP), TCP 80 (HTTP), TCP 443 (HTTPS) |
| GitHub OAuth App | Created at github.com/settings/developers |
| OpenRouter API key | From openrouter.ai |

## Step 1: AWS Setup

### EC2 Instance

1. Launch an EC2 instance (t3.small minimum, Amazon Linux 2023)
2. Allocate an **Elastic IP** and associate it to the instance
3. Configure **Security Group**:
   - Inbound: TCP 22 (your IP), TCP 80 (0.0.0.0/0), TCP 443 (0.0.0.0/0)
   - Outbound: all traffic

### DNS

Create two A records:

```
agents-orchestrator.com            → A → <Elastic IP>
monitoring.agents-orchestrator.com → A → <Elastic IP>
```

Verify with `dig agents-orchestrator.com` and `dig monitoring.agents-orchestrator.com`.

### GitHub OAuth App

1. Go to **github.com → Settings → Developer settings → OAuth Apps → New OAuth App**
2. Set:
   - **Homepage URL**: `https://agents-orchestrator.com`
   - **Authorization callback URL**: `https://agents-orchestrator.com/auth/github/callback`
3. Note the **Client ID** and **Client Secret**

## Step 2: EC2 Setup

SSH into the instance and run the setup script:

```bash
scp scripts/setup-ec2.sh ec2-user@<EC2_IP>:~/
ssh ec2-user@<EC2_IP>
chmod +x setup-ec2.sh && ./setup-ec2.sh
```

The script installs Docker, creates the app directory (`/opt/agent-orchestrator`), and generates `.env.prod` with random secrets.

## Step 3: Configure Environment

Edit the generated `.env.prod`:

```bash
nano /opt/agent-orchestrator/.env.prod
```

Fill in the values:

```env
ENVIRONMENT=production
BASE_URL=https://agents-orchestrator.com

# PostgreSQL (auto-generated, keep as-is)
POSTGRES_PASSWORD=<auto-generated>

# Auth
JWT_SECRET_KEY=<auto-generated>
OAUTH_CLIENT_ID=<from GitHub OAuth App>
OAUTH_CLIENT_SECRET=<from GitHub OAuth App>

# API Keys
OPENROUTER_API_KEY=<from openrouter.ai>

# Grafana (auto-generated)
GRAFANA_PASSWORD=<auto-generated>

# GitHub
GITHUB_USERNAME=<your GitHub username>
```

## Step 4: Deploy

### First deployment

```bash
cd /opt/agent-orchestrator
git clone <repo-url> .

# Start services (without SSL first — for certbot ACME challenge)
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d postgres redis dashboard grafana prometheus nginx
```

### Obtain SSL Certificates

```bash
# Request certificates for both domains
docker compose -f docker-compose.prod.yml run --rm certbot certonly \
  --webroot -w /var/www/certbot \
  -d agents-orchestrator.com -d monitoring.agents-orchestrator.com \
  --agree-tos --email your@email.com --non-interactive

# Restart Nginx to load the certificates
docker compose -f docker-compose.prod.yml restart nginx
```

### Start All Services

```bash
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d
```

### Verify

```bash
# Dashboard health check
curl -s https://agents-orchestrator.com/health

# Grafana
curl -s https://monitoring.agents-orchestrator.com/api/health

# All containers
docker compose -f docker-compose.prod.yml ps
```

## Step 5: GitHub Actions (CI/CD)

The deploy pipeline (`.github/workflows/deploy.yml`) auto-deploys on push to `main`.

### Required GitHub Secrets

| Secret | Value |
|--------|-------|
| `EC2_SSH_PRIVATE_KEY` | SSH private key for EC2 access |
| `EC2_HOST` | Elastic IP or domain name |

Set them in **repo → Settings → Secrets and variables → Actions**.

## SSL Certificate Renewal

Certificates auto-renew via the `certbot` container (checks every 12h). To manually renew:

```bash
docker compose -f docker-compose.prod.yml run --rm certbot renew
docker compose -f docker-compose.prod.yml restart nginx
```

## Monitoring

- **Grafana**: `https://monitoring.agents-orchestrator.com` (login: admin / `GRAFANA_PASSWORD`)
- **Prometheus**: `127.0.0.1:9090` (localhost only, access via SSH tunnel)

```bash
# Prometheus (raw metrics, SSH tunnel)
ssh -L 9090:localhost:9090 ec2-user@<EC2_IP>
# Open http://localhost:9090
```

## Security

| Layer | Protection |
|-------|-----------|
| **Transport** | TLS 1.2+, HSTS, strong ciphers |
| **Auth** | GitHub OAuth2 + JWT sessions, API keys (dashboard) |
| **Grafana auth** | Admin password, sign-up disabled |
| **Rate limiting** | 10 req/s per IP on dashboard (Nginx) |
| **Headers** | X-Frame-Options, X-Content-Type-Options, Referrer-Policy |
| **Secrets** | No secrets in code, `.env.prod` excluded from git |
| **Metrics** | `/metrics` blocked externally (403) |
| **CI/CD** | pip-audit, CodeQL, Trivy, TruffleHog scans |

## Troubleshooting

### Dashboard not loading

```bash
docker compose -f docker-compose.prod.yml logs dashboard --tail 50
docker compose -f docker-compose.prod.yml logs nginx --tail 50
```

### Grafana not loading

```bash
docker compose -f docker-compose.prod.yml logs grafana --tail 50
```

### SSL certificate issues

```bash
docker compose -f docker-compose.prod.yml run --rm certbot certificates
docker compose -f docker-compose.prod.yml run --rm certbot renew --force-renewal
docker compose -f docker-compose.prod.yml restart nginx
```

### Database connection

```bash
docker compose -f docker-compose.prod.yml exec postgres pg_isready -U orchestrator
docker compose -f docker-compose.prod.yml exec postgres psql -U orchestrator
```

### WebSocket not connecting

Check that the Security Group allows TCP 443 and that Nginx WebSocket config has long timeout (`proxy_read_timeout 86400`).

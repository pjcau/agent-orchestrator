# 11 - Data Persistence

## Overview

Paperclip uses Drizzle ORM with PostgreSQL for all persistent state. The schema has 55+ tables across 40+ migrations, reflecting the rich domain model.

## ORM Choice: Drizzle

Drizzle is a TypeScript-first ORM that generates SQL queries from a declarative schema. Unlike Prisma, it doesn't require code generation — the schema IS the type definition.

```typescript
// Example from issues service
const rows = await db
  .select()
  .from(issues)
  .where(and(
    eq(issues.companyId, companyId),
    eq(issues.status, "in_progress"),
  ))
  .orderBy(desc(issues.updatedAt));
```

## Migration Strategy

Migrations are generated by Drizzle Kit and stored as SQL files:

```
migrations/
├── 0000_initial.sql
├── 0001_add_agents.sql
├── ...
└── 0039_latest.sql
```

40+ migrations show rapid schema evolution. Auto-migration is supported via `PAPERCLIP_MIGRATION_AUTO_APPLY=true`.

## Embedded PostgreSQL

For local development, Paperclip bundles a PostgreSQL binary:

```typescript
import EmbeddedPostgres from "embedded-postgres";
// Starts Postgres on port 54329 with data in ~/.paperclip/postgres/
```

This eliminates Docker dependency for local dev. Data persists across restarts. Backup system with configurable intervals and retention.

## Key Patterns
- Drizzle ORM for type-safe SQL without code generation
- One table per file, clean exports from index
- 40+ migrations showing rapid iteration
- Embedded Postgres for zero-config local dev
- Auto-backup with configurable interval and retention

## Relevance to Our Project
We use in-memory storage with optional PostgreSQL. Paperclip's 55-table schema shows how rich a domain model becomes when you add budgets, approvals, plugins, and work products. The embedded Postgres pattern eliminates our Docker dependency. Drizzle's approach (schema-as-types, no codegen) is arguably better than SQLAlchemy's declarative base for type safety.
